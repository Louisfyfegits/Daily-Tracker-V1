# Daily Planner README

---

## Bundling

The browser can't load all the separate JS files and imports (especially firebase) directly, so we use esbuild to smash everything into one single file — dist/bundle.js. That's the file index.html actually loads. To bundle you have a few options, run these from the project folder:

- `npm install` — run this on a fresh clone to download node_modules (all the dependencies)
- `npm run build` — bundles everything once into dist/bundle.js
- `npm run watch` — same as build but watches for file changes and rebuilds automatically, use this while developing (why when i had the cmd closed did the css update, do css updates ignore this)

---

## Mobile vs Extension for developing

The mobile version will not update while developing as it runs through Netlify. It still sees data updates at the same time as the extension because they both use the same Firebase server. To get the mobile version to update you need to re-upload the relevant files (including dist/bundle.js) to Netlify manually. The extension updates automatically because it reads the files locally from your machine.

## JS Files

- `index.js` — main file, wires up all the event listeners and Firestore listeners
- `firebase.js` — sets up the Firebase connection and exports the database
- `utils.js` — timer helper functions (calculating elapsed time, running the interval)
- `data/days.js` — all Firestore logic for daily tasks, large tasks and daily counters
- `data/trackers.js` — all Firestore logic for timers and global counters
- `ui/render.js` — everything that touches the DOM to display data

---

## Other Files

**index.html** — the structure of the app, all the elements that JS hooks into. This is what the browser/extension actually opens.

**styles.css** — all the styling for the app.

**package.json** — tells Node what the project is, what dependencies it needs (firebase, esbuild), and defines the build/watch commands. This is how npm knows what to install and what `npm run build` means.

**package-lock.json** — auto-generated by npm, locks the exact versions of every dependency so installs are consistent. Don't edit this manually. Gets updated constantly by npm so expect it to change a lot, just commit it as is.

**manifest.json** — tells Chrome about the extension. The name, version, what icon to use, and what page to open as the popup. Only relevant when running as an extension, ignored everywhere else.

**icon.png** — the extension icon that shows in the Chrome toolbar.

**dist/bundle.js** — the bundled output file generated by esbuild. This is what index.html actually loads. Don't edit this manually, it gets overwritten every build.

**node_modules/** — all the downloaded dependencies (firebase, esbuild etc). Not on GitHub, regenerate with `npm install`.
